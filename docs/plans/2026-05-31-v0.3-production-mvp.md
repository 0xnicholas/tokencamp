# v0.3 — Production MVP 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** PostgreSQL 集成、DualCache、Hook 系统（限流+成本追踪）、管理 API、健康检查、Fallback。

**Architecture:** DualCache（LRU + Redis）前置 PG，Hook 链解耦横切关注点。限流和成本追踪作为 pre/post hook 插入请求路径。SpendLog 通过 Redis queue 异步批量写入 PG。API Key 唯一权威源迁移到 PG。

**Tech Stack:** SQLx, redis-rs, sha2, lru, uuid

**关联 Spec:** [docs/specs/2026-05-31-v0.3-production-mvp.md](../specs/2026-05-31-v0.3-production-mvp.md)

### Task 2: DualCache 实现

- [ ] **Step 1: 编写 cache/mod.rs**

```rust
// tokencamp-core/src/cache/mod.rs
pub trait CacheLayer: Send + Sync {
    async fn get(&self, key: &str) -> Option<String>;
    async fn set(&self, key: &str, value: &str, ttl: Duration);
    async fn del(&self, key: &str);
    async fn incr(&self, key: &str) -> Result<u32, String>;
    async fn expire(&self, key: &str, ttl: u64);
    async fn rpush(&self, key: &str, value: &str);
}

pub struct DualCache {
    in_memory: LruCache<String, (String, Instant)>,
    redis: Option<redis::aio::MultiplexedConnection>,
}

impl DualCache {
    pub fn new(capacity: usize, redis_conn: Option<MultiplexedConnection>) -> Self { ... }
}

impl CacheLayer for DualCache {
    // get: in-memory → Redis → None
    // set: in-memory + Redis SETEX
    // incr: Redis INCR (or Mutex<HashMap> for in-memory)
    // expire: Redis EXPIRE
    // rpush: Redis RPUSH
}
```

In-memory fallback for incr：用 `Mutex<HashMap<String, u32>>` 模拟原子计数。

```rust
async fn authenticate(key: &str, cache: &DualCache, db: &DbPool) -> Result<AuthContext> {
    let hash = sha2::Sha256::digest(key.as_bytes());
    let hash_hex = hex::encode(hash);
    
    // 1. 查缓存
    if let Some(cached) = cache.get(&format!("key:{}", hash_hex)).await {
        return deserialize_auth(&cached);
    }
    // 2. 查 PG
    if let Some(row) = db.find_key_by_hash(&hash_hex).await {
        let ctx = AuthContext { key_id: row.id.to_string(), ... };
        cache.set(&format!("key:{}", hash_hex), &serialize(&ctx), Duration::from_secs(300)).await;
        return Ok(ctx);
    }
    Err(401)
}
```

```rust
pub struct AuthContext {

pub enum HookError {
    #[error("rate limit exceeded: {0}")]
    RateLimitExceeded(String),
    #[error("hook internal error: {0}")]
    Internal(String),
}

#[async_trait]
pub trait ProxyHook: Send + Sync {
    async fn async_pre_call_hook(&self, request: &ChatRequest, auth: &AuthContext) -> Result<(), HookError>;
    async fn async_post_call_hook(&self, request: &ChatRequest, response: &ModelResponse, auth: &AuthContext);
    async fn async_on_error_hook(&self, request: &ChatRequest, error: &ProviderError, auth: &AuthContext);
}
```

- [ ] **Step 2: 在 main.rs 中按 YAML 构建 Hook 链**

```rust
fn build_hooks(config: &Config, cache: Arc<DualCache>) -> Vec<Box<dyn ProxyHook>> {
    let mut hooks: Vec<Box<dyn ProxyHook>> = Vec::new();
    for name in &config.hooks.enabled {
        match name.as_str() {
            "parallel_request_limiter" => hooks.push(Box::new(ParallelRequestLimiter::new(cache.clone()))),
            "cost_tracker" => hooks.push(Box::new(CostTracker::new(cache.clone()))),
            _ => eprintln!("Unknown hook: {}", name),
        }
    }
    hooks
}
```
