# v0.4 — Advanced Routing Strategies 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将 hardcoded `simple_shuffle` 重构为可插拔 `RoutingStrategy` trait，实现 5 种策略。

**Architecture:** RoutingStrategy trait 定义 `select_deployment` + `track_success` + `track_failure`。策略通过 YAML 启动时选择。Router 注入 `Box<dyn RoutingStrategy>`，请求后回调 track_*。

**Tech Stack:** 无新依赖

**关联 Spec:** [docs/specs/2026-05-31-v0.4-routing-strategies.md](../specs/2026-05-31-v0.4-routing-strategies.md)

---

### Task 1: RoutingStrategy trait 定义

- Create `tokencamp-core/src/router_strategy/mod.rs`

```rust
#[async_trait]
pub trait RoutingStrategy: Send + Sync {
    fn name(&self) -> &'static str;
    async fn select_deployment<'a>(
        &self, deployments: &'a [ModelEntry], request: &ChatRequest, cache: &dyn CacheLayer,
    ) -> Option<&'a ModelEntry>;
    async fn track_success(&self, deployment: &ModelEntry, response: &ModelResponse, duration_ms: Option<u64>);
    async fn track_failure(&self, deployment: &ModelEntry, error: &ProviderError);
}
```

- Register in `tokencamp-core/src/lib.rs`: `pub mod router_strategy;`
- Export: `pub use router_strategy::RoutingStrategy;`
- Build & commit

### Task 4: lowest_latency

- Create `lowest_latency.rs`
- Uses `cache` for latency storage (LPUSH + LTRIM on success)
- `select`: read all, average, pick smallest
- Build & commit

### Task 5: usage_based

- Create `usage_based.rs`
