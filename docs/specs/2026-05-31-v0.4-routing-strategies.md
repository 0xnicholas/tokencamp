## 2. RoutingStrategy trait

```rust
#[async_trait]
pub trait RoutingStrategy: Send + Sync {
    fn name(&self) -> &'static str;

    /// 从候选列表中选出一个 Deployment
    /// 调用前 Router 已过滤 cooldown + unhealthy
    async fn select_deployment<'a>(
        &self, deployments: &'a [ModelEntry], request: &ChatRequest, cache: &dyn CacheLayer,
    ) -> Option<&'a ModelEntry>;

    /// 调用成功后更新策略指标
    async fn track_success(&self, deployment: &ModelEntry, response: &ModelResponse, duration_ms: Option<u64>);

    /// 调用失败后更新策略权重
    async fn track_failure(&self, deployment: &ModelEntry, error: &ProviderError);
}
```

## 3. 四种策略

### 3.1 simple_shuffle（已有实现，提取为策略）

随机加权选择，无状态。

```
select: 随机选一个
track_success: no-op
track_failure: no-op
```

### 3.2 lowest_cost

按 model_info 价格排序，选最便宜。

```
select: 按 litellm_params.model_info.prompt_price 升序，取第一个
track_success: no-op
track_failure: no-op
```

### 3.3 lowest_latency

滑动窗口平均延迟，选最快。

**数据结构**：

```
Redis key: strategy:latency:{model}:{deployment_id} → List<f64> (最近 N 次延迟)
配置: latency_window_size = 10 (默认)
```

```
select: 读取所有 deployment 的平均延迟 → 选最小
track_success: LPUSH latency_value → LTRIM 到 window_size
track_failure: 延迟设为 INF 或增加惩罚分
```

无 Redis 时降级为 in-memory `HashMap<String, Vec<f64>>`。

### 3.4 usage_based

按 Deployment 当前负载选最空闲的。

**不同于** `ParallelRequestLimiter`（per API Key TPM）：此策略跟踪的是 **per Deployment** 的实时调用量。

```
Redis key: strategy:usage:{model}:{deployment_id}:tpm  (per deployment TPM)
           strategy:usage:{model}:{deployment_id}:rpm  (per deployment RPM)

select:
  1. 从 cache 读取每个 deployment 的当前 TPM/RPM
  2. 计算使用率 = max(current_tpm / tpm_limit, current_rpm / rpm_limit)
  3. 选使用率最低的

track_success: INCR tpm_counter by response.usage.total_tokens, INCR rpm_counter by 1 (带 TTL)
track_failure: no-op
```

### 3.5 tag_based

按 deployment 标签匹配请求标签。

```yaml
model_list:
  - model_name: gpt-5
    provider: openai
    model_info:
      tags: ["fast", "cheap"]
```

```
select: 从 request.extra["_tags"] 取请求标签 → 匹配 deployment tags → 过滤 → 选第一个
track_success: no-op
track_failure: no-op
```

## 4. Router 集成

```rust
pub struct Router {
    cooldown: Arc<CooldownManager>,
    strategy: Box<dyn RoutingStrategy>,
}

impl Router {
    pub fn new(cooldown: CooldownManager, strategy: Box<dyn RoutingStrategy>) -> Self { ... }

    pub async fn select_deployment<'a>(
        &self, model_name: &str, deployments: &'a [ModelEntry],
        fallbacks: &HashMap<String, Vec<String>>,
        cache: &dyn CacheLayer,
    ) -> Result<&'a ModelEntry, RouterError> {
        // 1. 过滤 cooldown
        let available = self.filter_cooldown(deployments).await;

        // 2. 策略选择
        if let Some(dep) = self.strategy.select_deployment(&available, request, cache).await {
            return Ok(dep);
        }

        // 3. Fallback（不变）
        // ...
    }

    pub async fn track_success(&self, dep: &ModelEntry, resp: &ModelResponse, duration_ms: Option<u64>) {
        self.strategy.track_success(dep, resp, duration_ms).await;
    }

    pub async fn track_failure(&self, dep: &ModelEntry, err: &ProviderError) {
        self.strategy.track_failure(dep, err).await;
    }
}
```

## 5. 配置

```yaml
router_settings:
  routing_strategy: "lowest_latency"
  latency_window_size: 10
```

ModelInfo 扩展字段：

```yaml
model_list:
  - model_name: gpt-5
    provider: openai
    model_info:
      prompt_price: 0.27
      completion_price: 1.10
      tpm: 100000       # [NEW] deployment TPM limit (usage_based)
      rpm: 1000          # [NEW] deployment RPM limit (usage_based)
      tags: ["fast"]     # [NEW] tag_based 匹配
```

启动时构造策略：

```rust
fn build_strategy(config: &Config, cache: Arc<dyn CacheLayer>) -> Box<dyn RoutingStrategy> {
    match config.router_settings.routing_strategy.as_str() {
        "lowest_cost" => Box::new(LowestCostStrategy::new()),
        "lowest_latency" => Box::new(LowestLatencyStrategy::new(cache, config.router_settings.latency_window_size)),
        "usage_based" => Box::new(UsageBasedStrategy::new(cache)),
        "tag_based" => Box::new(TagBasedStrategy::new()),
        _ => Box::new(SimpleShuffleStrategy::new()),
    }
}
```

## 6. 请求路径变化

```
chat.rs:
  resolve_provider → Router.select_deployment (策略选择)
  → handler.complete (调用)
  → 成功: Router.track_success (策略更新指标)
  → 失败: Router.track_failure (策略更新权重)
```

| 标准 | 指标 |
|------|------|
