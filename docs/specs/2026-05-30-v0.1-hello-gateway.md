# Spec: v0.1 — Hello Gateway

**版本**: v0.1  
**主题**: 单模型透传，证明 Rust + Axum 技术栈在 LLM Gateway 场景下的可行性  
**状态**: Frozen  
**日期**: 2026-05-30  
**关联 ADR**: [001](../adr/001-rust-axum-tech-stack.md), [002](../adr/002-gateway-core-dual-layer.md), [003](../adr/003-provider-adapter-pattern.md), [008](../adr/008-openai-compatible-api.md)

---

## 1. 目标

构建一个最小可用单元的 LLM API Gateway：

- 客户端发送 OpenAI 格式的 ChatCompletion 请求
- Gateway 将请求转换为 Provider 原生格式，转发到 OpenAI API，将响应转回 OpenAI 格式后返回
- 用最简单的认证（YAML 中配置 API Key）保护 Gateway
- 项目骨架为后续版本打好 crate 边界和 trait 接口

**证明**：Rust + Axum 栈在构建 LLM Gateway 时：
- 请求路径延迟可接受（目标是 < 单次 Provider 调用 + 5ms 开销）
- Provider 适配器 traint 接口设计合理（新增 OpenAI 以外的 Provider 只需实现 trait）

### 2.2 请求路径

```
客户端
  │  POST /v1/chat/completions
  │  Header: Authorization: Bearer sk-xxx
  ▼
gateway/src/routes/chat.rs
  │  1. 读取 state.config (AppState)
  │  2. 验证 API Key → KeyAuth (extractor)
  │  3. 反序列化 ChatRequest (serde)
  │  4. 调用 core_handler.complete(request, provider)
  ▼
tokencamp-core/src/handler.rs
  │  5. provider.transform_request(request) → Provider 原生 body
  │  6. HTTP POST → Provider API (reqwest)
  │  7. provider.transform_response(status, headers, body) → ModelResponse
  ▼
gateway/src/routes/chat.rs
  │  8. 序列化 ModelResponse → JSON
  │  9. 返回 200 + JSON body
  ▼
客户端
```

## 3. 接口定义

### 3.1 ProviderConfig trait

```rust
// tokencamp-core/src/provider/mod.rs

use async_trait::async_trait;
use reqwest::{header::HeaderMap, StatusCode};
use bytes::Bytes;

#[async_trait]
pub trait ProviderConfig: Send + Sync {
    /// 将 OpenAI ChatRequest 转换为 Provider 原生请求 body (JSON Value)
    async fn transform_request(
        &self,
        request: &ChatRequest,
        api_key: &str,
        headers: &mut HeaderMap,
    ) -> Result<(reqwest::Method, String, serde_json::Value), ProviderError>;

    /// 将 Provider 原生响应转换为 ModelResponse
    /// body 是已读取的完整 bytes（v0.1 非流式）
    async fn transform_response(
        &self,
        status: StatusCode,
        headers: &HeaderMap,
        body: Bytes,
    ) -> Result<ModelResponse, ProviderError>;

    /// Provider 的基础 URL
    fn base_url(&self) -> &str;
}
```

**设计要点**：
- `transform_request` 返回 `(Method, URL path, body)`——v0.1 只有 POST，但 trait 预留 URL path 灵活性
- `api_key` 作为参数传入而非存在 ProviderConfig 内部——ProviderConfig 本身不持有密钥，由调用方注入
- `body: Bytes` 而非 `reqwest::Response`——Response body 只能消费一次（Rust 所有权），在 handler 层统一 read 后传给 transform

### 3.3 HttpHandler

```rust
// tokencamp-core/src/handler.rs

pub struct HttpHandler {
    client: reqwest::Client,
}

impl HttpHandler {
    pub fn new() -> Self {
        Self {
            client: reqwest::Client::builder()
                .timeout(Duration::from_secs(60))
                .build()
                .expect("Failed to create HTTP client"),
        }
    }

    pub async fn complete(
        &self,
        request: &ChatRequest,
        provider: &dyn ProviderConfig,
        api_key: &str,
    ) -> Result<ModelResponse, ProviderError> {
        let mut headers = HeaderMap::new();
        let (method, path, body) = provider.transform_request(request, api_key, &mut headers).await?;
        
        let url = format!("{}{}", provider.base_url(), path);
        
        let response = self.client
            .request(method, &url)
            .headers(headers)
            .json(&body)
            .send()
            .await?;
        
        let status = response.status();
        let resp_headers = response.headers().clone();
        let resp_body = response.bytes().await?;
        
        provider.transform_response(status, &resp_headers, resp_body).await
    }
}
```

## 4. OpenAI Provider 实现

```rust
// tokencamp-core/src/provider/openai.rs

pub struct OpenAiConfig {
    base_url: String,
}

impl OpenAiConfig {
    pub fn new(base_url: Option<String>) -> Self {
        Self {
            base_url: base_url.unwrap_or_else(|| "https://api.openai.com".to_string()),
        }
    }
}

#[async_trait]
impl ProviderConfig for OpenAiConfig {
    fn base_url(&self) -> &str {
        &self.base_url
    }

    async fn transform_request(
        &self,
        request: &ChatRequest,
        api_key: &str,
        headers: &mut HeaderMap,
    ) -> Result<(Method, String, Value), ProviderError> {
        headers.insert(
            "Authorization",
            format!("Bearer {}", api_key).parse().unwrap(),
        );
        headers.insert("Content-Type", "application/json".parse().unwrap());

        let body = serde_json::to_value(request)?;
        Ok((Method::POST, "/v1/chat/completions".to_string(), body))
    }

    async fn transform_response(
        &self,
        status: StatusCode,
        _headers: &HeaderMap,
        body: Bytes,
    ) -> Result<ModelResponse, ProviderError> {
        if !status.is_success() {
            let error: Value = serde_json::from_slice(&body)?;
            return Err(ProviderError::UpstreamError {
                status: status.as_u16(),
                message: error["error"]["message"].as_str().unwrap_or("unknown").to_string(),
            });
        }
        
        let response: ModelResponse = serde_json::from_slice(&body)?;
        Ok(response)
    }
}
```

**因为 v0.1 只有 OpenAI，且 OpenAI 的请求/响应格式即规范格式，所以 `transform_request` 几乎不做转换（只加 header），`transform_response` 直接反序列化。** 但 trait 接口必须完整实现，这是 v0.2 接入 Anthropic（需要格式转换）时的前置条件。

### 5.3 启动入口

```rust
// gateway/src/main.rs

use axum::{Router, routing::post};
use std::net::SocketAddr;
use std::sync::Arc;

mod config;
mod auth;
mod routes;

#[derive(Clone)]
pub struct AppState {
    pub config: Arc<Config>,
    pub handler: Arc<HttpHandler>,
}

impl AppState {
    /// 根据 model_name 查找对应 provider 及其 api_key。
    /// v0.1: 直接映射，不涉及路由选择。
    /// 返回 (ProviderConfig, api_key)
    pub fn resolve_provider(
        &self,
        model_name: &str,
    ) -> Result<(Box<dyn ProviderConfig>, String), AppError> {
        let entry = self.config.model_list.iter()
            .find(|m| m.model_name == model_name)
            .ok_or(AppError::ModelNotFound {
                model: model_name.to_string(),
            })?;

        let provider = self.config.providers.get(&entry.provider)
            .ok_or(AppError::ProviderNotFound {
                provider: entry.provider.clone(),
            })?;

        let base_url = provider.base_url.clone();
        let api_key = provider.api_key.clone();

        // v0.1 只有 OpenAI，直接构造
        let provider_config: Box<dyn ProviderConfig> = match entry.provider.as_str() {
            "openai" => Box::new(OpenAiConfig::new(base_url)),
            other => return Err(AppError::ProviderNotFound {
                provider: other.to_string(),
            }),
        };

        Ok((provider_config, api_key))
    }
}

#[tokio::main]
async fn main() {
    let config = Arc::new(config::load("config/default.yaml").expect("Failed to load config"));
    let handler = Arc::new(HttpHandler::new());
    
    let state = AppState { config, handler };
    
    let app = Router::new()
        .route("/v1/chat/completions", post(routes::chat::chat_completions))
        .with_state(state);
    
    let addr = SocketAddr::from(([0, 0, 0, 0], 3000));
    println!("Tokencamp v0.1 listening on {}", addr);
    
    axum::serve(tokio::net::TcpListener::bind(addr).await.unwrap(), app)
        .await
        .unwrap();
}
```

### 5.4 认证 Extractor

```rust
// gateway/src/auth.rs

use axum::{
    extract::FromRequestParts,
    http::{request::Parts, StatusCode},
};
use crate::AppState;

pub struct KeyAuth;

impl FromRequestParts<AppState> for KeyAuth {
    type Rejection = (StatusCode, &'static str);

    async fn from_request_parts(
        parts: &mut Parts,
        state: &AppState,
    ) -> Result<Self, Self::Rejection> {
        let auth_header = parts
            .headers
            .get("authorization")
            .and_then(|v| v.to_str().ok())
            .ok_or((StatusCode::UNAUTHORIZED, "missing authorization header"))?;

        let key = auth_header
            .strip_prefix("Bearer ")
            .ok_or((StatusCode::UNAUTHORIZED, "invalid authorization format"))?;

        if state.config.auth.api_keys.iter().any(|k| k == key) {
            Ok(KeyAuth)
        } else {
            Err((StatusCode::UNAUTHORIZED, "invalid api key"))
        }
    }
}
```

### 5.5 Chat 路由

```rust
// gateway/src/routes/chat.rs

use axum::{extract::State, Json};
use tokencamp_core::{ChatRequest, ModelResponse};

use crate::{AppState, AppError};

pub async fn chat_completions(
    State(state): State<AppState>,
    _auth: KeyAuth,                    // v0.1: 认证通过才进入路由
    Json(request): Json<ChatRequest>,
) -> Result<Json<ModelResponse>, AppError> {
    // v0.1: 不支持流式。如果客户端请求流式，返回错误
    if request.stream.unwrap_or(false) {
        return Err(AppError::StreamNotSupported);
    }

    let (provider_config, api_key) = state.resolve_provider(&request.model)?;
    let response = state.handler.complete(&request, provider_config.as_ref(), &api_key).await?;
    Ok(Json(response))
}
```

**v0.1 不涉及模型路由**：`model_list` 中一个 model_name 只对应一个 provider。直接映射，不选。

## 6. 错误处理

### 6.1 Core 层错误

```rust
// tokencamp-core/src/provider/mod.rs

#[derive(Debug, thiserror::Error)]
pub enum ProviderError {
    #[error("upstream error: status={status}, message={message}")]
    UpstreamError { status: u16, message: String },
    
    #[error("HTTP error: {0}")]
    HttpError(#[from] reqwest::Error),
    
    #[error("serialization error: {0}")]
    SerializationError(#[from] serde_json::Error),
    
    #[error("timeout")]
    Timeout,
}
```

### 6.2 Gateway 层错误

```rust
// gateway/src/routes/mod.rs (或独立 error.rs)

use axum::{response::{IntoResponse, Response}, http::StatusCode, Json};
use tokencamp_core::ProviderError;

#[derive(Debug, thiserror::Error)]
pub enum AppError {
    #[error("provider error: {0}")]
    Provider(#[from] ProviderError),

    #[error("model not found: {model}")]
    ModelNotFound { model: String },

    #[error("provider not found: {provider}")]
    ProviderNotFound { provider: String },

    #[error("streaming not supported in v0.1")]
    StreamNotSupported,
}

impl IntoResponse for AppError {
    fn into_response(self) -> Response {
        let (status, message) = match &self {
            AppError::Provider(ProviderError::UpstreamError { status: s, message: m }) => {
                (StatusCode::from_u16(*s).unwrap_or(StatusCode::INTERNAL_SERVER_ERROR), m.clone())
            }
            AppError::Provider(ProviderError::Timeout) => {
                (StatusCode::GATEWAY_TIMEOUT, "upstream timeout".to_string())
            }
            AppError::Provider(_) => {
                (StatusCode::BAD_GATEWAY, "upstream error".to_string())
            }
            AppError::ModelNotFound { model } => {
                (StatusCode::BAD_REQUEST, format!("model '{}' not found", model))
            }
            AppError::ProviderNotFound { provider } => {
                (StatusCode::INTERNAL_SERVER_ERROR, format!("provider '{}' not configured", provider))
            }
            AppError::StreamNotSupported => {
                (StatusCode::BAD_REQUEST, "streaming is not supported in this version".to_string())
            }
        };
        
        let body = serde_json::json!({ "error": { "message": message } });
        (status, Json(body)).into_response()
    }
}
```

## 9. 不做

以下明确不在 v0.1 范围内，避免范围膨胀：

- ❌ 流式响应 (SSE)
- ❌ 多 Provider (仅 OpenAI)
- ❌ Redis / 任何外部缓存
- ❌ 数据库 (配置全在 YAML)
- ❌ Hook 系统
- ❌ 路由策略 (model_name → provider 直接映射)
- ❌ 成本追踪 / 限流 / 日志
- ❌ 健康检查
- ❌ 管理 API
- ❌ 任何高级路由
