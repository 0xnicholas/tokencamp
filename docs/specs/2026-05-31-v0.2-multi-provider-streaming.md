# Spec: v0.2 — Multi-Provider + Streaming

**版本**: v0.2  
**主题**: Anthropic 适配（转换 + Passthrough）、SSE 流式、Cooldown/Retry 容错  
**状态**: Frozen  
**日期**: 2026-05-31  
**关联 ADR**: [003](../adr/003-provider-adapter-pattern.md), [008](../adr/008-openai-compatible-api.md), [011](../adr/011-resilience-patterns.md), [014](../adr/014-streaming.md)

---

## 1. 目标

在 v0.1 单模型透传基础上增加：

- **Anthropic Provider 适配**：两个路径——
  - `POST /v1/chat/completions`（OpenAI 格式 → Anthropic Messages API 转换）
  - `POST /v1/messages`（Anthropic 原生格式 Passthrough，直通）
- **SSE 流式响应**：`stream: true` 支持
- **Router + simple-shuffle**：多 Deployment 时随机选择
- **Cooldown（Redis 最小版）+ Retry**：3 次指数退避重试 + Redis cooldown 熔断
- **`GET /v1/models`**：返回可用模型列表

### 不做

- Google Gemini / Azure（仅 Anthropic）
- DualCache 完整版（仅 cooldown 用 Redis）
- API Key 缓存、限流（v0.3）
- Fallback 策略
- 任何高级路由策略（lowest-latency 等）

### 2.2 ProviderConfig trait 扩展

pub trait ProviderConfig: Send + Sync {
    // --- v0.1 ---
    async fn transform_request(&self, request: &ChatRequest, api_key: &str, headers: &mut HeaderMap)

## 3. AnthropicConfig 实现

### 3.1 双模式

```rust
pub struct AnthropicConfig {
    mode: AnthropicMode,
    base_url: String,
}

pub enum AnthropicMode {
    Chat,       // OpenAI → Anthropic Messages 转换
    Messages,   // Passthrough
}
```

### 3.2 Chat 模式：格式转换

| OpenAI 字段 | Anthropic 字段 | 转换逻辑 |
|------------|---------------|---------|
| `messages[role=system]` | `system` (顶层数组) | 提取所有 system message，放入顶层 `system` |
| `messages[role=user/assistant]` | `messages` | 直传 role + content |
| `model` | `model` | 映射（见 3.4） |
| `temperature` | `temperature` | 直传 |
| `max_tokens` | `max_tokens` | 直传，默认 1024 |
| `stream` | `stream` | 直传 |
| 认证 header | `x-api-key` + `anthropic-version` | Bearer → x-api-key |

**System message 处理是关键差异**：Anthropic 的 `system` 是顶层字段而非 messages 数组的一部分。v0.2 只支持 text system content（不支持 tool_use 等复杂结构）。

### 3.3 Messages 模式：Passthrough

在 `transform_request` 中 `is_passthrough() = true` 时：
- 请求体原样转发（不做格式转换）
- 只添加 `x-api-key`、`anthropic-version`、`Content-Type` headers

**Passthrough 响应同样原样返回**——不经过 `transform_response`。`HttpHandler` 在 `is_passthrough()` 时跳过格式转换，直接返回原始 `Bytes`，由路由层序列化为 `Json<Value>`：

```rust
// HttpHandler 新增方法
pub async fn complete_raw(
    &self, request, provider, api_key,
) -> Result<Bytes, ProviderError> {
    // ... 同 complete() 前部分 ...
    // 跳过 transform_response，直接返回 bytes
    Ok(resp_body)
}

// messages.rs
pub async fn anthropic_messages(...) -> Result<Json<Value>, AppError> {
    let raw = state.handler.complete_raw(&request, provider_config.as_ref(), &api_key).await?;
    let value: Value = serde_json::from_slice(&raw)?;
    Ok(Json(value))
}
```

`ChatRequest` 需支持存储原生 JSON：

```rust
#[derive(Debug, Deserialize, Serialize)]
pub struct ChatRequest {
    pub model: String,
    pub messages: Vec<Message>,
    #[serde(default)]
    pub temperature: Option<f32>,
    #[serde(default)]
    pub max_tokens: Option<u32>,
    #[serde(default)]
    pub stream: Option<bool>,
    #[serde(flatten)]
    pub extra: HashMap<String, Value>,  // [NEW] 透传 Anthropic 原生字段
}
```

### 3.4 模型名映射

配置中维护映射表：

```yaml
model_list:
  - model_name: claude-sonnet-4-5
    provider: anthropic
    litellm_params:
      model: claude-sonnet-4-5-20250929
```

Gateway 对外暴露 `claude-sonnet-4-5`，内部映射到 Anthropic 的完整 model ID。

### 3.5 Anthropic SSE → OpenAI SSE

```
Anthropic SSE 事件格式:
  event: content_block_delta
  data: {"type": "content_block_delta", "index": 0, "delta": {"type": "text_delta", "text": "Hello"}}

转换后 OpenAI SSE:
  data: {"id":"...","object":"chat.completion.chunk","choices":[{"index":0,"delta":{"content":"Hello"}}]}
```

关键 Anthropic 事件映射：

| Anthropic 事件 | 产生的 OpenAI Chunk |
|---------------|-------------------|
| `message_start` | `delta.role = "assistant"` |
| `content_block_start` | (忽略) |
| `content_block_delta` (text) | `delta.content = text` |
| `content_block_delta` (tool_use) | (v0.2 忽略) |
| `message_delta` | `finish_reason` + `usage` |
| `message_stop` | `done = true` |

pub async fn anthropic_messages(
    State(state): State<AppState>,
    _auth: KeyAuth,

    Ok(Json(value))

## 5. Cooldown + Retry

### 5.1 Retry 策略

在 `HttpHandler` 层实现：

```rust
pub async fn complete(
    &self, request, provider, api_key,
) -> Result<ModelResponse, ProviderError> {
    let mut attempt = 0;
    loop {
        match self.try_call(request, provider, api_key).await {
            Ok(resp) => return Ok(resp),
            Err(e) if is_retryable(&e) && attempt < 3 => {
                attempt += 1;
                let delay = 0.5 * 2u64.pow(attempt) + rand::random::<u64>() % 100;  // 指数退避 + jitter (rand crate)
                tokio::time::sleep(Duration::from_millis(delay)).await;
                // 如果是 429，优先用 Provider 返回的 Retry-After
            }
            Err(e) => return Err(e),
        }
    }
}

fn is_retryable(e: &ProviderError) -> bool {
    matches!(e, ProviderError::UpstreamError { status: 429, .. }
               | ProviderError::UpstreamError { status: 500..=599, .. }
               | ProviderError::Timeout)
}
```

### 5.2 Cooldown

```rust
// gateway/src/router/cooldown.rs
pub struct CooldownManager {
    redis: redis::aio::MultiplexedConnection,
}

impl CooldownManager {
    pub async fn mark(&self, deployment_id: &str, ttl: u64) -> Result<()> {
        self.redis.set_ex(format!("deployment:{}:cooldown", deployment_id), "1", ttl).await
    }

    pub async fn is_cooling_down(&self, deployment_id: &str) -> Result<bool> {
        self.redis.exists(format!("deployment:{}:cooldown", deployment_id)).await
    }
}
```

触发条件：连续失败 `allowed_fails`（配置项，默认 3）次。

Cooldown 配置：

```yaml
router_settings:
  num_retries: 3
  retry_after: 0.5
  allowed_fails: 3
  cooldown_time: 30  # seconds
```

### 5.3 Router.select_deployment()

```
1. 查找所有匹配 model_name 的 Deployment
2. 过滤 cooldown 中的 Deployment
3. 从剩余候选中 simple_shuffle 选一个
4. 如果没有可用 Deployment → 返回错误
```

## 6. Redis 集成

v0.2 仅引入最小 Redis 集成：

- **依赖**：`redis = { version = "0.27", features = ["tokio-comp", "aio"] }`
- **用途**：仅 cooldown 状态，单 key 格式 `deployment:{id}:cooldown`
- **可选**：无 Redis 时降级为内存 cooldown（单实例可用，多实例不一致）

```yaml
# config/default.yaml 新增
redis:
  url: ${REDIS_URL}   # 可选，不配置则降级内存模式
```
